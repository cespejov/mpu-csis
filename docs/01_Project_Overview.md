# Project Overview

## 1. Project Name

Child Services Information System (CSIS)

## 2. Background

Non-profit organization managing child welfare programs often rely on manual records or disconnected spreadsheets. This creates data inconsistency, reporting delays, and limited visibility into child health, education, and socio-economic conditions.

## 3. Purpose

The purpose of this system is to digitalize and centralize child-related socio-economic, health, and education records to improve operational efficiency, data accuracy, and reporting capabilities.

## 4. Objectives

- Centralize child data management
- Track health and nutrition records
- Track household socio-economic indicators
- Monitor education progress
- Enable structured reporting
- Improve data integrity and security

## 5. Scope

### In Scope

- Child registration and profile management
- Guardian management and relationship mapping
- Health record tracking (nutrition, lifestyle, status)
- Education tracking
- Household registration and tracking
- Socio-economic indicators per household
- Role-based user authentication
- Reporting module

### Out of Scope

- Mobile application
- External government system integration
- Payment processing
- Biometric integration

## 6. Target Users

- Administrators
- Social Workers
- Education Coordinators

## 7. High-Level Features

- CRUD operations for core entities
- Role-based access control
- Relational database with enforced constraints
- Modular frontend architecture
- MVC-based backend structure

## 8. Technology Stack

### Frontend

- HTML5
- CSS3
- Vanilla JavaScript (Functional/Modular pattern)

### Backend

- PHP (Object-Oriented, MVC architecture)

### Database

- MySQL (Normalized relational design)

## 9. Expected Benefits

- Holistic view of children within their household context
- Improved data accuracy
- Improved socio-economic risk assessment
- Reduced manual errors
- Better visibility into child welfare indicators

## 10. SDLC Approach

This project follows a structured Software Development Life Cycle (SDLC) including:

- Requirement analysis
- System architecture design
- Database modeling (ER design)
- Implementation using MVC architecture
- Test planning and validation
- Deployment planning
- Maintenance strategy
